# Sylphx — messaging kit

## Elevator — EN

Sylphx is the AI-native platform that replaces your frankenstack. One SDK for services and deploy, real per-project isolation, hibernate when idle, AI gateway included — plus open tools that prove the engineering.

## Primary tagline (locked v1)

**Stop stitching tools together. Ship with one platform.**

## Secondary lines

| Use | Line |
|-----|------|
| OSS / broad | Empowering developers to build the future with AI |
| Infra narrative | Building AI agent infrastructure |
| Category | AI-native PaaS · developer platform |

## Pillars

1. **Unify the stack** — one SDK, one key, one dashboard story  
2. **Real isolation** — dedicated Postgres + cache per project  
3. **Economic idle** — hibernate to near-zero; wake in seconds  
4. **AI first-class** — gateway as a service  
5. **Open craft** — MCP, libraries, public engineering  

## Objection frames

| Objection | Frame |
|-----------|--------|
| “I’ll just use Vercel + Supabase.” | Fine until the tenth vendor and the third auth story. We sell back your integration time. |
| “Is isolation real?” | Per-project Postgres and managed cache — not shared soft tenancy cosplay. |
| “Another AI wrapper?” | AI is one service among twenty-plus. The product is the platform. |
| “Lock-in?” | Typed SDK and portable engineering culture; evaluate export paths honestly in product docs. |

## Prefer / avoid

| Prefer | Avoid |
|--------|--------|
