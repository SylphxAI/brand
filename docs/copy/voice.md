# Sylphx — voice & samples

## Voice

**Clear, technical, confident. Developer-first wit — never clownish. Precise about infrastructure.**

## Principles

1. Prefer concrete nouns: SDK, Postgres, hibernate, gateway, deploy.  
2. Show the trade: time and vendors saved, not vibes.  
3. Open-source voice can be warmer; platform voice stays sharp.  
4. Never invent compliance certifications.  
5. Humor: dry, rare, earned.

## Sample — changelog (EN)

Per-project cache now hibernates with the database path. Idle projects stop paying for warmth they don’t use; first request wakes the pair. No API shape change — just fewer surprises on the bill.

## Sample — OSS README blurb (EN)

Part of the Sylphx open toolchain — built for agents and humans who need reliable infrastructure primitives, not demo-ware.

## Sample — error state (EN)

Project is hibernated. First API call will wake compute and data. This usually takes seconds.

## Avoid

Crypto-bro tone, “disrupting cloud”, fake benchmark bragging, renaming the product SylphxAI in customer UI.
